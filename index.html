HTML

  • The head element contains information about the webpage.
  • The meta element is used to specify character set, page descriptioin, keywords, author of the document, and viewport settings and is nested in the head section.
  • The title element is nested in the head section and names our document.
  • The body element represents the visible content shown to the user.
  • The headings elements are numbered h1-h6 and represent different levels of section headings with "h1" being the most prominent and "h6" being the least prominent.
  • The division element is a generic block-level container.
  • The span element is a generic inline container.
  • The anchor element creates a hyperlink.
  • The line break element creates a line break where it is placed.

Git

  • git status: checks what branch we are currently on
  • git add [file]: adds a file as it looks now to your next commit (stage)
  • git reset [file]: unstage a file while retaining the changes in the working directory
  • git diff: diff of what is changed but not staged
  • git diff --staged: diff of what is staged but not yet committed
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot
  • git branch: list your branches. a * will appear next to the currently active branch
  • git branch [branch-name]: create a new branch at the current commit
  • git checkout -b branch-name: creates a new branch and switches to it
  • git merge [branch]: merge the specified branch's history into the current one
  • git log: show all commits in the current branch's history
